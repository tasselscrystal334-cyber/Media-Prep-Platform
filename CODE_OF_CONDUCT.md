# Code of Conduct

## Purpose

Set collaboration expectations for MediaPrep Studio contributors and maintainers.

## Overview

The project expects professional, respectful, inclusive, and safety-minded collaboration across engineering, documentation, design, support, and AI-assisted development.

## Architecture

This document aligns with the MediaPrep Studio layered architecture: presentation, application, services, core, infrastructure, codec engine, media engine, and hardware integration.

## Workflow

Discuss technical decisions with evidence, document outcomes in the repository, and escalate maintainership or safety concerns through project owners.

## Dependencies

No runtime dependencies.

## Configuration

No configuration.

## Example

Review code and documents by focusing on behavior, risk, and maintainability rather than personal judgment.

## Known Limitations

This document describes the intended architecture and current command-line implementation. Desktop, cloud, and enterprise features may be staged behind roadmap milestones.

## Future Improvements

Expand this document when the subsystem receives a new module, public API, UI surface, or deployment contract.
