# Roadmap

## Purpose

Define the long-term product direction for Loom.

## Version Plan

| Version | Theme | Outcome |
| --- | --- | --- |
| v0.1 | Repository Foundation | GitHub-ready repository, documentation, tests, and CLI skeleton. |
| v0.2 | Core Framework | Project model, configuration, logging, errors, and service boundaries. |
| v0.3 | Media Validation | Rule engine, profiles, status model, and reports. |
| v0.4 | Metadata Engine | Stable metadata extraction, cache, and search model. |
| v0.5 | FFmpeg Integration | FFprobe, FFmpeg deep analysis, and FFplay preview foundations. |
| v0.6 | Batch Processing | Job queue, task status, cancellation, and resumable operations. |
| v0.7 | Quality Control | Live event, codec, canvas, hash, and integrity QC workflows. |
| v0.8 | FFplay Preview | Operator preview, frame stepping, audio checks, and safe playback. |
| v0.9 | Codec Expansion | HAP, NotchLC, ProRes, DNxHR, EXR, TIFF, and hardware codec coverage. |
| v1.0 | Public Release | Stable CLI, reports, packaging, docs, examples, and support policy. |
| v1.5 | Node Workflow | Node-based media server workflows and project handoff manifests. |
| v2.0 | Distributed Processing | Worker nodes, shared cache, NAS workflow, and remote processing. |
| v3.0 | Cloud Collaboration | Project sync, review links, audit history, and cloud metadata services. |
| v4.0 | AI Assistant | Assisted troubleshooting, rules authoring, naming review, and report summaries. |
| v5.0 | Enterprise Platform | Licensing, SSO, plugin marketplace, deployment tooling, and support SLAs. |
